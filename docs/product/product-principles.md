# PigeonYang Cover Product Principles

## Product Form
This product is an Agent-native cover design system, not a standalone UI-first app.

Codex is the primary runtime. Skills provide design reasoning. A local MCP should later provide deterministic project, file, validation, and metrics operations. GPT Image 2 remains the preferred image generation engine through the native Agent environment.

## First User
The first user is Yang himself. The product should optimize Yang's real publishing workflow before serving other users.

## First Platform
The first production platform is WeChat public account cover design. Yang's content pipeline is article-first: successful articles can later become videos or Xiaohongshu posts.

## Core Objective
The first objective is to improve open rate. Brand consistency matters, but it must not override the single article's click promise.

## Operating Rules
- Strategy comes before image generation.
- Each task must produce three differentiated cover directions before final generation.
- Cover copy can be proposed by the Agent, but final wording requires Yang's approval.
- Yang's character identity is mandatory for first-version covers.
- The character must remain in a refined anime / illustration style, not drift into photorealism.
- Private production projects and metrics must stay outside the product git repo.
- Product repo stores tools, skills, standards, specs, and reusable templates.

## Design Standard
The preferred visual direction is high-click knowledge-media cover design: large hooks, strong contrast, high information clarity, and enough polish to avoid cheap marketing aesthetics.

## Privacy Boundary
Private cover production records belong under:

```text
J:\PigeonYang\cover-style-distiller\cover-projects
```

They must not be committed to the product repo.
