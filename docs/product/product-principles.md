# PigeonYang Cover Product Principles

## Product Form
This product is an Agent-native cover design skill router, not a standalone
UI-first app and not a generic cover prompt template.

Codex is the primary runtime. Distilled creator child skills provide design
engines. The mother skill routes article drafts to suitable engines and enforces
workflow, privacy, identity, platform, and firewall rules. A local MCP should
later provide deterministic project, file, validation, and metrics operations.
GPT Image 2 remains the preferred image generation engine through the native
Agent environment.

## First User
The first user is Yang himself. The product should optimize Yang's real publishing workflow before serving other users.

## First Platform
The first production platform is WeChat public account cover design. Yang's content pipeline is article-first: successful articles can later become videos or Xiaohongshu posts.

## Core Objective
The first objective is to improve open rate. Brand consistency matters, but it must not override the single article's click promise.

## Core Product Bet
The main product advantage is not "AI makes a cover." It is that the system can
accumulate many distilled creator-cover engines and choose the right engine for
the user's draft.

The product should therefore optimize routing quality:

- Which child skill best fits this article's hook?
- Which engine should be avoided because it would distort the article?
- What visual evidence can the user compare before choosing?
- How does the final PigeonYang cover keep the selected engine's strengths
  without naming or copying a public creator?

## Operating Rules
- Strategy comes before image generation.
- Each task must first route the article to suitable distilled child-skill
  engines.
- Each task must produce three visual direction references before final
  generation.
- Cover copy can be proposed by the Agent, but final wording requires Yang's approval.
- Yang's character identity is mandatory for first-version covers.
- The character must remain in a refined anime / illustration style, not drift into photorealism.
- Private production projects and metrics must stay outside the product git repo.
- Product repo stores tools, skills, standards, specs, and reusable templates.

## Design Standard
The preferred visual direction is selected by the router from the child-skill
library. "High-click knowledge-media cover design" is a baseline, not the
engine. The actual engine may be thesis authority, maker-proof, documentary
evidence, visible stakes, transformation, aesthetic identity, or another
distilled creator pattern.

Do not bypass the child-skill library unless no existing engine fits.

## Privacy Boundary
Private cover production records belong under:

```text
J:\PigeonYang\cover-style-distiller\cover-projects
```

They must not be committed to the product repo.
