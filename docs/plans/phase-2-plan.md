# PLAN: Phase 2

## Goal
Validate the WeChat cover workflow with real articles and turn subjective design
feedback plus publish metrics into concrete product improvements.

## Phase 2 Deliverable
Run 3-5 real WeChat article cover projects end to end:

1. create a private project,
2. produce three directions,
3. approve one direction and exact copy,
4. generate with GPT Image 2,
5. verify dimensions and identity,
6. record post-publish metrics,
7. convert repeated friction into script or skill changes.

## Work Sequence
1. Pick the first real article draft or title.
2. Run the private project workflow without changing the product code.
3. Record every manual interruption, unclear decision, failed prompt, bad output,
   and post-generation correction.
4. After 3-5 runs, classify friction into workflow, prompt, identity, dimension,
   metrics, or MCP/tooling issues.
5. Patch only repeated or high-cost problems.

## Acceptance
- At least 3 private projects exist under the private project root.
- Each project has directions, approval, execution packet, final prompt, metrics,
  and review notes.
- At least one project includes a generated output with verified dimensions.
- No private project content is committed to the product repo.
- Product changes are based on observed friction, not speculative features.
